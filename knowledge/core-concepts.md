# Core Concepts

## Purpose

This document groups the foundational BCH concepts that BCH Defender should be able to explain clearly.

## Main Concept Areas

### 1. What BCH Is

BCH should be explained as:

- Bitcoin Cash
- peer-to-peer electronic cash
- a decentralized payment network
- a practical money and payment system rather than only a speculative asset

### 2. Wallets and Control

BCH Defender should clearly distinguish:

- BCH from a wallet
- wallet interface from network behavior
- app state from blockchain state
- self-custody from custodial dependence

### 3. Keys, Seed Phrases, and Recovery

The knowledge layer should support explanations of:

- seed phrases
- private keys
- restoration
- recovery limits
- why backup matters

### 4. Addresses and Formats

BCH Defender should be able to explain:

- BCH receiving addresses
- CashAddr
- legacy BCH address formats
- why address-format confusion matters operationally

### 5. Transaction Visibility and Confirmation

Core explanations should distinguish:

- visible payment
- zero-conf
- block confirmation
- on-chain evidence versus app screenshots

### 6. UTXO-Based Understanding

The knowledge layer should support simple explanations of:

- UTXOs
- how BCH transactions are constructed conceptually
- why wallet balances and transaction structure behave the way they do

## Reusable Teaching Rule

A good BCH knowledge answer should:

1. define the concept clearly
2. show one practical implication
3. clarify one common confusion if useful
