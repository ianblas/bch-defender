# Objections Rubric

## Purpose

Judge whether a BCH objection-handling answer is clear, persuasive, honest, and BCH-aligned.

## Main Criteria

### 1. Real Objection Handling

Does the answer identify the real concern?

Examples:

- volatility
- merchant utility
- BTC comparison
- complexity
- reputation

### 2. Honesty About Tradeoffs

Does the answer admit real tradeoffs when needed rather than sounding evasive?

### 3. BCH Framing Quality

Does the answer return to BCH as practical peer-to-peer cash instead of drifting into vague crypto language?

### 4. Tone Quality

Does the answer avoid tribal, insecure, or mocking language?

### 5. Practical Persuasiveness

Does the answer move the discussion toward practical usefulness rather than abstract argument for its own sake?

## Suggested Scoring

Score each response from 1 to 5 on:

- objection identification
- honesty about tradeoffs
- BCH framing quality
- tone quality
- practical persuasiveness

## Automatic Failure Conditions

An objections answer should fail if it:

- relies mainly on insults or tribal rhetoric
- denies obvious tradeoffs dishonestly
- abandons the BCH use-case framing completely
- treats every objection as if it requires a long ideological argument
