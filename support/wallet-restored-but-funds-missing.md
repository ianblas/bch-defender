# Wallet Restored but Funds Missing

## Summary

Sometimes a user restores a BCH wallet and does not see the expected balance.

That does not automatically mean the BCH is gone.

In many cases, the problem is wallet setup, restore flow, or transaction scanning.

## Why It Matters

Users can panic quickly when a restored wallet shows no funds.

A good BCH support answer should reduce panic and focus on verification.

## What to Verify

### 1. Confirm the seed phrase was restored correctly

A single mistake can produce a different wallet.

### 2. Confirm the restore happened in the correct BCH wallet or account flow

The user may be in the wrong asset, wrong account, or wrong app flow.

### 3. Check whether the wallet supports rescanning or refreshing transaction history

Sometimes the BCH is on-chain but not yet visible in the app view.

### 4. Check the relevant BCH addresses on-chain if possible

If the funds are visible on-chain, the issue may be local wallet state rather than actual loss.

## Step-by-Step Guidance

### 1. Slow down and verify the restore details

Do not begin by assuming theft or permanent loss.

### 2. Confirm the seed and the BCH wallet context

Make sure the user restored the intended wallet in a BCH-compatible flow.

### 3. Refresh or rescan if the wallet allows it

A missing app balance does not always mean the BCH is missing on-chain.

### 4. Distinguish between app visibility and on-chain reality

If the BCH is still visible on-chain, the next step is wallet troubleshooting, not panic.

## Common Causes

- wrong seed phrase
- wrong wallet flow or account
- wallet has not rescanned yet
- user is checking the wrong balance view

## Common Mistakes

- assuming the BCH is gone immediately
- confusing app state with blockchain state
- skipping on-chain verification
- restoring into the wrong wallet context

## Short Answer

If a restored BCH wallet shows no funds, first verify the seed, wallet context, and transaction history. If the BCH still exists on-chain, the main problem is often restore or wallet state rather than actual loss.
