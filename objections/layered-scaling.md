# Why Layered Scaling Cannot Be the Main Monetary Model

## Summary

Layered scaling can be useful in some contexts, but from a BCH perspective it cannot replace a strong, scalable base layer for peer-to-peer electronic cash.

The core argument is simple: if the main chain is not robust, usable, and economically well-incentivized, then building higher layers on top does not solve the problem. It only moves the problem upward into systems that are usually more fragile, more custodial, more regulated, and less aligned with Bitcoin's original incentive structure.

This is why BCH does not reject layers in principle. It rejects the idea that Bitcoin should become intentionally weak at the base layer and then depend on higher layers for ordinary payments.

## Core Idea

A layered system only works well if the base layer is strong.

If the base layer is intentionally constrained, then the higher layers inherit that weakness. From a BCH perspective, a healthy design looks like this:

- layer 1 remains useful for normal peer-to-peer payments
- layer 1 remains scalable and economically active
- higher layers add extra functionality without parasitizing the base layer
- higher layers do not replace the core monetary use of the chain

The key BCH claim is that **Bitcoin must scale at the base layer first** if it wants to remain peer-to-peer electronic cash.

## The Base Layer Must Be Strong First

The central claim of the episode is that all higher layers sit on top of layer 1. If layer 1 is weak, the whole structure becomes weak. The hosts compare it to trying to build a tower on sand: the foundation must be strong enough to support everything built above it. ŅłĆfileciteŅłéturn12file1Ņłü

From a BCH perspective, that means layer 1 must be:

- scalable
- usable
- economically active
- well-incentivized
- robust under real user demand

If it is not, then talking about layer 2 or layer 3 as the main answer becomes misleading.

## Why BCH Says Layer 1 Must Handle Payments

BCH treats layer 1 as the natural place for everyday peer-to-peer payments.

The episode states this directly: BCH's layer 1 is for transacting, for daily payments, and for supporting peer-to-peer electronic cash. Those everyday transactions are what help incentivize the base layer. ŅłĆfileciteŅłéturn12file0Ņłü

This matters because:

- economic activity on layer 1 helps reward miners
- direct use strengthens the chain's security model
- broad payment usage keeps the network aligned with its original purpose
- higher layers can exist without replacing the base use case

From a BCH perspective, when the base layer stops being the place where people actually use the money, the incentive model starts breaking.

## Good Layers vs Bad Layers

The episode draws a distinction between layers that complement the base layer and layers that parasitize it.

### A good higher layer

A good higher layer:

- adds functions the base layer does not natively provide well yet
- does not compete with normal layer-1 payments
- does not undermine miner incentives
- can even send some economic value back to the base chain

The hosts describe a good layer as one that handles a different class of demand rather than stealing the base layer's core role. ŅłĆfileciteŅłéturn12file0Ņłü

### A bad higher layer

A bad higher layer:

- competes with the base layer for ordinary payments
- weakens the incentive model of layer 1
- pushes users toward custody or semi-custody
- centralizes liquidity or control
- becomes easier to regulate and censor

From the BCH perspective, Lightning on BTC is the clearest example of a bad higher layer in practice. The episode calls it a parasitic layer because it competes with the base layer for peer-to-peer cash transactions instead of complementing it. ŅłĆfileciteŅłéturn12file1Ņłü

## Why Lightning Is Criticized in This Framework

The episode gives several reasons why Lightning is considered a bad answer when used as BTC's main payment model.

### 1. It competes with the base layer

If the base layer is supposed to be peer-to-peer cash, and then a second layer takes over that same role, the incentive structure is distorted.

The hosts argue that Lightning tries to capture the transactions that should be happening directly on the base chain, leaving only leftovers for miners. ŅłĆfileciteŅłéturn12file1Ņłü

### 2. It pushes users toward custody

### 3. It tends toward centralization

## Why BCH Rejects ŌĆ£Scale by Layers OnlyŌĆØ

The BCH view is not that layers are always wrong.

The BCH view is that **scaling by layers only** is the wrong monetary model for Bitcoin.

Why?

Because a peer-to-peer cash system must preserve direct, cheap, reliable payments at the base layer.

If it does not, then:

- the chain becomes less useful as money
- users move toward intermediaries
- the base incentive model weakens
- higher layers become more important than the actual chain
- the system starts recreating the structure Bitcoin was meant to escape

## The Incentive Problem

This is ultimately an incentives argument.

The episode repeatedly emphasizes that layer 1 must remain active and used because the whole system depends on the incentives of the base layer. If that layer is weakened, everything above it becomes weaker too. ŅłĆfileciteŅłéturn12file0Ņłéturn12file1Ņłü

From a BCH perspective:

- many cheap transactions are healthy
- broad user activity is healthy
- direct merchant and user payments are healthy
- miner incentives should be tied to real network usage

A model that intentionally limits layer 1 and then asks layers above it to save the system is backwards.

## Can Layers Still Be Useful on BCH?

Yes.

The episode explicitly leaves room for useful higher layers. The key condition is that they must not attack or replace the base role of peer-to-peer cash. ŅłĆfileciteŅłéturn12file0Ņłü

That means a higher layer may be useful when it:

- provides specialized functionality
- supports contracts or applications not yet native to layer 1
- serves a different type of demand than everyday payments
- coexists with a strong base layer instead of replacing it

This is why BCH's position is not anti-layer. It is anti-parasitic-layer.

## Short Answer

If someone asks, ŌĆ£Can Bitcoin scale mainly through layers?ŌĆØ a BCH-style answer would be:

Only if the base layer remains strong, scalable, and economically healthy. A layered system built on top of a weak or intentionally constrained base layer becomes more fragile, more custodial, more centralized, and more vulnerable to regulation.

## Conclusion

From a BCH perspective, the correct model is not ŌĆ£make layer 1 weak and push payments upward.ŌĆØ

The correct model is:

- keep layer 1 strong
- keep layer 1 scalable
- keep layer 1 useful for ordinary payments
- allow higher layers to add extra functionality without replacing peer-to-peer cash

That is why BCH rejects the idea that Bitcoin should scale mainly by layers.

Layers can be helpful, but they are not a substitute for a healthy base chain.
